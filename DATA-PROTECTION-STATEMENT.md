# Correctover Data Protection & Availability Statement

> **Purpose**: This document describes Correctover's data protection measures, data availability policy, and the rationale behind the proprietary/public data boundary.
>
> **Status**: v1.0 (2026-07-04)
> **Repository**: Correctover/standards

## Preamble: Why Data Protection Is the Standard

Agent Conformance testing operates at the intersection of production telemetry, provider infrastructure, and autonomous execution traces. Unlike traditional software benchmarks, conformance data for LLM-based agents contains:

- **Real production failure patterns** from third-party LLM providers, captured under implicit operational confidentiality
- **Identity and authorization traces** that, if exposed, could enable adversarial exploitation of verified systems
- **Proprietary verification logic** whose exposure would invalidate the test itself — analogous to revealing examination criteria to candidates before assessment

This is not a choice between "open" and "closed." It is a structural requirement: **a conformance standard whose test data is fully public cannot detect adversarial adaptation.** The data protection framework is therefore not a commercial afterthought — it is a technical prerequisite for the standard's validity.

The classification system below is designed to maximize scientific reproducibility while preserving the conditions under which the standard remains technically meaningful.

## 1. Data Classification

Correctover classifies its conformance testing data into four tiers:

| Tier | Classification | Public? | Access Method |
|------|---------------|---------|---------------|
| **Tier 1** | Specification & Schema | ✅ Public | CC BY 4.0, GitHub, Zenodo DOI |
| **Tier 2** | Aggregate Statistics | ✅ Public | METHODOLOGY documents, Zenodo DOI |
| **Tier 3** | Anonymized Sample Data | 🔒 Under NDA | Data Use Agreement (academic) |
| **Tier 4** | Raw Production Data | 🔒 Proprietary | Not available |

## 3. What Is Available Under NDA (Tier 3)

### 3.1 Anonymized Sample Traces (Up to 500 records)
**Available to**: Academic researchers, peer reviewers, independent implementers

**Requirements**:
- Signed Data Use Agreement (see DATA-USE-AGREEMENT-v1.md)
- Institutional affiliation or independent researcher status
- Brief description of intended use

**What is provided**:
- Up to 500 anonymized traces in JSON format
- 325 failure mode classification table
- Aggregate statistics by test group
- Benchmark methodology document

### 3.2 Purpose
The Tier 3 access program enables:
- Independent verification of statistical claims
- Implementation of compatible conformance systems
- Academic research and publication
- Peer review validation

---

## 4. What Is Proprietary (Tier 4)

### 4.1 Raw Production Data
- **20,071 production traces** — contain provider-specific production telemetry
- **Full synthetic dataset** (29,929 traces) — generated by proprietary engine
- **Flywheel rule set** (88 rules) — core technology
- **Conformance runner implementation** — closed-source SDK component

### 4.2 Rationale for Non-Disclosure
1. **Competitive protection**: These assets constitute Correctover's core intellectual property and primary technological contribution
2. **Provider trust**: Production traces contain telemetry from third-party LLM providers under implicit operational confidentiality
3. **Technology integrity**: Disclosure of the verification engine's internal logic would compromise the specification's independence as a governance tool. Conformance testing requires that the verifier's implementation remain independent of the systems it verifies
4. **Standard integrity**: The specification's value depends on Correctover maintaining independent verification capability

**Resolution**:
1. The methodology document has been revised to include a formal **Reproducibility Statement** (Section 4) that clearly distinguishes:

## 7. Commitment to Reproducibility

Correctover is committed to making the conformance specification independently verifiable to the maximum extent possible without compromising:
1. Core technology (flywheel rules, runner implementation)
2. Provider trust (production telemetry)
3. Verifier independence (the specification's governance authority depends on the verifier remaining distinct from the systems it evaluates)

The combination of:
- **Public specification** (Tier 1) — enables compatible implementations
- **Public statistics** (Tier 2) — enables claim verification
- **NDA sample data** (Tier 3) — enables academic validation
- **Independent verification ecosystems** (PHI-OMEGA Runtime, preaction-governance-conformance) — demonstrate that the specification can be independently verified without access to Correctover's proprietary code

...provides a robust reproducibility framework that balances openness with sustainability.
